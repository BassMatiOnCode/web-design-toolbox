Sticky Row-Group Headers

Row group headers add vertical structure to a large table. Ideally, they consist of a single cell stretching over the entire table width and remains in place when the table is scrolled horizontally. In practice, we need two cells because scrolling takes precedence over stickyness if the cell is larger than the container window:

<thead><tr class="rowgroup-header">
	<th colspan="3">Group 1</th>
	<td colspan="6"></td>
</tr></thead>

This way, the TD cell can slide under the TH cell when the table is scrolled to the left.

If the group header text is wider than a single column, it is possible to assign more columns to the TH cell (3 in this example), as long as their width is less than the container window width.

The stylesheet code for the row-group header should now look familar: 

table.sticky-headers .rowgroup-header {
	position : sticky ;
	top : 22px ;
	left : 0px ;
	z-index : 2 ;
	background-color : white ;
	text-align : left ;
	}
table.sticky-headers .rowgroup-header th {
	border-right-style : none ;
	}

Again, a background color is required to prevent the data cells from shining through. The right border was removed to give the illusion of a single cell in the header row.
