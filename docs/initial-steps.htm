Initial Steps to Set Up a Web Site

Step 1 – Create the Infrastructure

  1. Extract contents of toolbox-inc-min.zip to the site root folder. This creates the index.htm and toc.htm files, as well as the inc, fnt and img subfolders with a minimum set of files to create a working infrastructure.
  2. Create or customize one or more HTML page template files.
  3. Optional: Integrate page templates into the "new" command of Windows Explorer (or your system's file system explorer) to avoid copy and paste duplication.
  4. Create the web site root folder.
  5. Edit inc/header.js and .css files to define the logo image, header text and colors. More details on how to achieve this will follow.
  6. Edit index.htm and supply some initial content.

Step 2 – Add Pages

To add a new page,…

  1. Create a copy of the page template and rename it to something meaningful, such as page-2.htm.
  2. Edit page-2.htm to give it some meaningful content. Put the bulk of the content inside the DIV id="content" element.
  3. Edit toc.htm, and add a new LI with a nested A element inside:
<ul>
<li><a href="/index.htm">Home</a></li>
<li><a href="/page-2.htm">Page 2</a></li>
</ul>

The HEAD Element in the Page Template

The HEAD element in the page template deserves a few words. It is shown in listing 3:

<!DOCTYPE html><html lang="en-US"><head>
<meta charset="utf-8">
<meta name="viewport" content="width=device-width, initial-scale=1.0">
<meta name="template-info" file-name="empty.htm" version="2022-01-17" editor="usp" />
<meta name="author" content="usp" />
<meta name="creation-date" content="2021-08-04" />
<meta name="editor" content="usp" />
<meta name="change-date" content="2021-08-04" />
<meta name="version" content="1" />
<meta name="categories" content="Programming > Web Design > Toolbox" />
<meta name="tags" content="HTML, JavaScript, CSS, standard, page, structure" />
<meta name="description" content="A standard page has a number of standard elements: A header panel, a toolbar, a content container, a footer, and a navigation-side-panel, which is hidden initially." />
<title>Web Design Toolbox - Standard Page Structure</title>
<link rel="stylesheet" href="/inc/page.css" />
</head>

Without going into the details, there are some meta elements that are related to the specific page, such as author, change date, et cetera. It is a good habit to supply meaningful values there. Some of these fields are used to generate content for the footer element at the bottom of the page. The description and title fields are used by the browser and search engines and should always be filled with meaningful information. Additionally, the description field is used to generate the page abstract paragraph in the body, if present. Others, such as tags and categories are more related to entire web site and your personal search tools. These can safely be deleted from the template if you don't need them. Likewise, feel free to add your own meta elements to the template.

Step 3 – Preview the Page

This step is optional. To preview the page in your web browser, …

  1. Double-click on page-2.htm in Windows Explorer.

You should see the formatted web page, but without header, footer, toolbar and navigation panel. This is okay, because the page init script will not be executed over the file:// protocol for security reasons. You can do proofreading, tweak the layout until you are satisfied with the results. Just remember to refresh the page in the browser after changes in the editor program.

Step 4 – Live Test on a Development Server

This requires to run small and simple web server, like Deno file-server or node http-server. No server-side processing is required, no CGI, no PHP. The server only must serve static files correctly over the http:// protocol, then all scripts should be working in the browser now.

  1. Run the script start-webserver.bat in the website root directory.
  2. Note the port number the server is listening on.
  3. Open a web browser, and type localhost:4507/page-2.htm into the address bar. Use the port number as given by the server. You should see the finished web page with a header, toolbar and footer.
  4. Click on the navigate button to open the naviation side-panel.
  5. Click the links to verify that they are working correctly.

Remember to refresh the page to see the changes made in the editor.

Step 5 – Deployment to a Production Server

If you are developing in a professional environment, you certainly have a separate production web server. Publishing on a production server usually requires some form of copying the (modified) files from the development machine to the production server. Contact your administrators for details and access permissions.

A private person may want to omit this step and develop the pages directly on the production server. This is perfectly okay. To prevent your customers from watching your progress, make shure not to add a link to the new page(s) to the toc.htm file. You can enter the page url explicitly into your browser's address box to test the page(s). When you are satisfied, just update the toc file(s) to make the new pages go online.
